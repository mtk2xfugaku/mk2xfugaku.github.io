What is a compiler ?

A compiler is a software tool used to convert a high level language into a low level machine language which can be executed on a computer. Often when somone is talking about compilation or a compiled programming language they are refering to a language like C/C++/Rust but languages like Python, Javascript are also compiled into a low level bytecode representation which can then be executed on an interpreter (a virtual machine) like CPython, V8, JVM.

A simple hello world program written in C 

   #include <stdio.h>

   int main() {
       printf("hello world");
   }

and compile as

$gcc -S main.c

and you will get the compiled assembly representation of your hello world program.


	.file	"main.c"
	.text
	.section	.rodata
.LC0:
	.string	"hello world"
	.text
	.globl	main
	.type	main, @function
main:
.LFB0:
	.cfi_startproc
	pushq	%rbp
	.cfi_def_cfa_offset 16
	.cfi_offset 6, -16
	movq	%rsp, %rbp
	.cfi_def_cfa_register 6
	movl	$.LC0, %edi
	movl	$0, %eax
	call	printf
	movl	$0, %eax
	popq	%rbp
	.cfi_def_cfa 7, 8
	ret
	.cfi_endproc
.LFE0:
	.size	main, .-main
	.ident	"GCC: (GNU) 12.3.1 20230508 (Red Hat 12.3.1-1)"
	.section	.note.GNU-stack,"",@progbits

Why is compiler important ?

Compilers were developed to improve how we represent computation (algorithms) and to decrease software (software was not a term back in the 1940s and 50s) complexity. Assembly language was actually developed as a high level language over binary representation used in the earlier systems. In today's standards writing assembly language is considered bad since compilers are better than the programmer (which is what we will see in this book) at generating fast and correct code. Although there are some niche instances where the programmer with assembly is better than the compiler itself, mostly in very low level programming and in computational kernel development (BLAS type stuff).

Compilers are the most studied area of computer science and the parsing methods used in compiler construction is the most well established area in theoratical computer science. These days most effort is put into the backend of the compiler (the transformation and analysis phases). Compilers are also very complex tools. I would compilers to be the most complex software out there.
